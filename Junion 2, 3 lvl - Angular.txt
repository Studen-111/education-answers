1. Создание проекта и рабочего пространства (CLI, отличия).

     Команда Ангулар CLI ng new my-workspace --create-application=false создает новый проект. Вы разрабатываете приложения в контексте рабочего пространства Ангуляра. Рабочее пространство может содержать файлы одного или нескольких проектов. Проект это набор файлов которые содержат отдельное приложение или общедоступную библиотеку.
     Проект, созданный с помощью ng new, содержит набор исходных файлов, ресурсов и файлов конфигурации, необходимых для разработки и тестирования приложения с помощью интерфейса командной строки. Проекты также могут быть созданы с помощью команд ng generate application и ng generate library.

2. JIT и AOT компиляция Angular приложения.

     JIT - Скомпилируйте TypeScript как раз вовремя для его выполнения.
     - Скомпилирован в браузере.
     - Каждый файл компилируется отдельно.
     - Нет необходимости строить после изменения кода и перед перезагрузкой страницы браузера.
     - Подходит для местного развития.

     AOT - Компиляция TypeScript на этапе сборки.
     - Компилируется самой машиной через командную строку ng build --prod (быстрее).
     - Весь код скомпилирован вместе, встраивая HTML/CSS в скрипты.
     - Нет необходимости развертывать компилятор (в половину меньше веса).
     - Более безопасный, исходный источник не разглашается.
     - Подходит для производственных сборок.

4. Component, Directive, Pipe, Service:
     a) Что это такое?
     b) Их декоратоты, метаданные, создание с помощью CLI (с флагами).
     c) Виды директив.
     d) Отличие компонента от директивы.
     
     a) Component - обособленная часть функционала со своей логикой, HTML-шаблоном и CSS-стилями. Directive - в Ангуляре применяется для определения собственных директив для привязки кастомного поведения к элементам в DOM. Pipe - инструмент для форматирование данных перед их отображением. Service - это объект без состояния и предоставляет некоторые очень полезные функции. Эти функции могут быть вызваны из любого компонента Angular, такого как контроллеры, директивы и т.д. Это помогает разделить веб-приложение на небольшие различные логические блоки, которые можно использовать повторно.

     @Injectable
     Чтобы указать, что сервис может использоваться в других сервисах, к классу сервиса применяется декоратор Injectable. Данный декоратор гарантирует, что встроенный механизм внедрения зависимостей сможет создать объект этого класса и передать его в качестве зависимости в другой объект (в другой сервис или компонент).
     providedIn?: Type<any> | 'root' | 'platform' | 'any' | null.
     
     @NgModule может содержать следующие параметры:
     providers - список инжектируемых объектов, которые добавляются в этот модуль
     declarations - компоненты, директивы и пайпы, принадлежащие этому модулю
     imports - модули, которые экспортируются декларируемыми и доступны в шаблоне этого модуля
     exports - компоненты, директивы и пайпы, которые объявлены декларируемыми, и могут быть им пользованы в шаблоне любого компонента, которые принадлежит NgModule импортирующему их.
     entryComponent - компилируемые компоненты при определении NgModule, для динамической загрузки в view.
     bootstrap - компоненты, которые загружаются при загрузке этого модуля, автоматически добавляются в entryComponent.
     schemas - набор схем, объявляющих разрешенные элементы в MgModules
     id - имя или путь, уникальный идентификатор этого NgModule в getModuleFactory. Если не заполнять - не будет там зарегистрирован.
     jit - если true, то этот модуль будет пропущен компилятором AOT и всегда будет компилироваться JIT.

     c) Структурные - для изменения структуры DOM-дерева. Атрибутивные - меняют внешний вид или поведение элемента DOM-дерева к которому они применяются.

5. Зачем нужно отписываться от Observable’ов? Как это делать?

     Если мы подпишемся на поток, он останется открытым и будет вызываться всякий раз, когда в него передаются значения из любой части приложения, до тех пор, пока он не будет закрыт с помощью вызова unsubscribe.
     Если подписка не закрыта, callback-функция будет непрерывно вызываться, что приведет к серьезной утечке памяти и проблемам с производительностью.
