1. Создание проекта и рабочего пространства (CLI, отличия).

     Команда Ангулар CLI ng new my-workspace --create-application=false создает новый проект. Вы разрабатываете приложения в контексте рабочего пространства Ангуляра. Рабочее пространство может содержать файлы одного или нескольких проектов. Проект это набор файлов которые содержат отдельное приложение или общедоступную библиотеку.
     Проект, созданный с помощью ng new, содержит набор исходных файлов, ресурсов и файлов конфигурации, необходимых для разработки и тестирования приложения с помощью интерфейса командной строки. Проекты также могут быть созданы с помощью команд ng generate application и ng generate library.

2. JIT и AOT компиляция Angular приложения.

     JIT - Скомпилируйте TypeScript как раз вовремя для его выполнения.
     - Скомпилирован в браузере.
     - Каждый файл компилируется отдельно.
     - Нет необходимости строить после изменения кода и перед перезагрузкой страницы браузера.
     - Подходит для местного развития.

     AOT - Компиляция TypeScript на этапе сборки.
     - Компилируется самой машиной через командную строку ng build --prod (быстрее).
     - Весь код скомпилирован вместе, встраивая HTML/CSS в скрипты.
     - Нет необходимости развертывать компилятор (в половину меньше веса).
     - Более безопасный, исходный источник не разглашается.
     - Подходит для производственных сборок.

5. Зачем нужно отписываться от Observable’ов? Как это делать?

     Если мы подпишемся на поток, он останется открытым и будет вызываться всякий раз, когда в него передаются значения из любой части приложения, до тех пор, пока он не будет закрыт с помощью вызова unsubscribe.
     Если подписка не закрыта, callback-функция будет непрерывно вызываться, что приведет к серьезной утечке памяти и проблемам с производительностью.

6. HostBinding, HostListener - что это и зачем нужно?

     HostBinding позволяет связать обычное свойство класса со свойством элемента, к которому применяется директива.
     Декоратор @HostListener позволяет связать события DOM и методы директивы. В частности, в декоратор передается название события, по которому будет вызываться метод.
     
     В Angular есть четыре формы привязки данных:     
     - Interpolation - привязка элемента DOM к значению компонента (односторонняя). В двойных фигурных скобках указывается выражение, к которому идет привязка: {{выражение}}. Например: <h1>Добро пожаловать {{name}}!</h1> 
     - Property, Attribute - привязка свойства элемента DOM к значению компонента (односторонняя). Например: <input type="text" [value]="name" />
     - Event - привязка метода компонента к событию в DOM (генерация события в DOM вызывает метод на компоненте)(односторонняя). Например: <button (click)="addItem(text, price)">Добавить</button>
     - Two-way - двусторонняя привязка, когда элемент DOM привязан к значению на компоненте, при этом изменения на одном конце привязки сразу приводят к изменениям на другом конце. Например: <input [(ngModel)]="name" placeholder="name"> 
     - Привязка к атрибуту элемента html
     - Привязка к классу CSS

     a) Что это? Преимущества и недостатки
     b) В чём проблема кода? Как исправить (2 способа)?
     …
     todos$ = this.todosService.requestAll(); // Request todos from the server
     …
     <div *ngIf=”todos$ | async”>
       <h1>Total: {{ (todos$ | async).length }}
       <div *ngFor=”let todo of (todos$ | async)”>
         {{ todo.title }}
       </div>
     </div>
     
     a) AsyncPipe позволяет получить результат асинхронной операции. AsyncPipe отслеживает объекты Observable и Promise и возвращает полученное из этих объектов значение. При получении значения AsyncPipe сигнализирует компоненту о том, что надо проверить изменения. Если компонент уничтожается, то AsyncPipe автоматически отписывается от объектов Observable и Promise, что устраняет возможные утечки памяти.
     b)

          <h1>Total: {{ todos.length }}</h1>

10. Что такое Observable?
     a) Hot и cold observable
     b) Отличие от Promise
     
     Observable – асинхронный паттерн проектирования, в котором данные представлены в виде потока событий. Потоки можно «переиспользовать» – комбинировать, фильтровать, подобно массивам, используя специальные методы – операторы.
     a) Cold Observable: 
        • Не рассылает объекты, пока на него не подписался хотя бы один подписчик; 
        • Если observable имеет несколько подписчиков, то он будет рассылать всю последовательность объектов каждому подписчику.
        Hot Observable:
        • Рассылает объекты, когда они появляются, независимо от того есть ли подписчики; 
        • Каждый новый подписчик получает только новые объекты, а не всю последовательность.
        
        b) Promise обрабатывает одно событие, когда асинхронная операция завершается успешно или с ошибкой.
        - Observable похож на поток и позволяет передавать ноль или более событий, когда для каждого события вызывается callback.
        - Observable также имеет преимущество перед обещанием быть отмененным
        - Observable предоставляет такие операторы,как map,forEach, reduce, ... и другие операторы для работы с массивами.
