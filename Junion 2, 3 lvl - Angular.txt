1. Создание проекта и рабочего пространства (CLI, отличия).

     Команда Ангулар CLI ng new my-workspace --create-application=false создает новый проект. Вы разрабатываете приложения в контексте рабочего пространства Ангуляра. Рабочее пространство может содержать файлы одного или нескольких проектов. Проект это набор файлов которые содержат отдельное приложение или общедоступную библиотеку.
     Проект, созданный с помощью ng new, содержит набор исходных файлов, ресурсов и файлов конфигурации, необходимых для разработки и тестирования приложения с помощью интерфейса командной строки. Проекты также могут быть созданы с помощью команд ng generate application и ng generate library.

2. JIT и AOT компиляция Angular приложения.

     JIT - Скомпилируйте TypeScript как раз вовремя для его выполнения.
     - Скомпилирован в браузере.
     - Каждый файл компилируется отдельно.
     - Нет необходимости строить после изменения кода и перед перезагрузкой страницы браузера.
     - Подходит для местного развития.

     AOT - Компиляция TypeScript на этапе сборки.
     - Компилируется самой машиной через командную строку ng build --prod (быстрее).
     - Весь код скомпилирован вместе, встраивая HTML/CSS в скрипты.
     - Нет необходимости развертывать компилятор (в половину меньше веса).
     - Более безопасный, исходный источник не разглашается.
     - Подходит для производственных сборок.

3. Router и ActivatedRoute - отличия, для чего используются.

     Router.
     Мы можем переходить по ресурсам внутри приложения с помощью ссылок. Но также Angular поддерживает программную навигацию. То есть программным образом из любого места приложения мы можем перейти к любому ресурсу. Для этого применяется сервис Router, который определен в пакете "angular/router" и который передается в компоненты через механизм dependency injection.
     
     ActivateRoute.
     Для получения параметров маршрута нам необходим специальный сервис ActivatedRoute. Он содержит информацию о маршруте, в частности, параметры маршрута, параметры строки запроса и прочее. Он внедряется в приложение через механизм dependency injection, поэтому в конструкторе мы можем получить его.
     Свойство snapshot хранит состояние маршрута, а состояние маршрута содержит переданные параметры.

     Если мы подпишемся на поток, он останется открытым и будет вызываться всякий раз, когда в него передаются значения из любой части приложения, до тех пор, пока он не будет закрыт с помощью вызова unsubscribe.

6. HostBinding, HostListener - что это и зачем нужно?

     HostBinding позволяет связать обычное свойство класса со свойством элемента, к которому применяется директива.
     Декоратор @HostListener позволяет связать события DOM и методы директивы. В частности, в декоратор передается название события, по которому будет вызываться метод.
     
     В Angular есть четыре формы привязки данных:     
     - Interpolation - привязка элемента DOM к значению компонента (односторонняя). В двойных фигурных скобках указывается выражение, к которому идет привязка: {{выражение}}. Например: <h1>Добро пожаловать {{name}}!</h1> 
     - Property, Attribute - привязка свойства элемента DOM к значению компонента (односторонняя). Например: <input type="text" [value]="name" />
     - Event - привязка метода компонента к событию в DOM (генерация события в DOM вызывает метод на компоненте)(односторонняя). Например: <button (click)="addItem(text, price)">Добавить</button>
     - Two-way - двусторонняя привязка, когда элемент DOM привязан к значению на компоненте, при этом изменения на одном конце привязки сразу приводят к изменениям на другом конце. Например: <input [(ngModel)]="name" placeholder="name"> 
     - Привязка к атрибуту элемента html
     - Привязка к классу CSS

10. Что такое Observable?
     a) Hot и cold observable
     b) Отличие от Promise
     
     Observable – асинхронный паттерн проектирования, в котором данные представлены в виде потока событий. Потоки можно «переиспользовать» – комбинировать, фильтровать, подобно массивам, используя специальные методы – операторы.
     a) Cold Observable: 
        • Не рассылает объекты, пока на него не подписался хотя бы один подписчик; 
        • Если observable имеет несколько подписчиков, то он будет рассылать всю последовательность объектов каждому подписчику.
        Hot Observable:
        • Рассылает объекты, когда они появляются, независимо от того есть ли подписчики; 
        • Каждый новый подписчик получает только новые объекты, а не всю последовательность.
        
        b) Promise обрабатывает одно событие, когда асинхронная операция завершается успешно или с ошибкой.
        - Observable похож на поток и позволяет передавать ноль или более событий, когда для каждого события вызывается callback.
        - Observable также имеет преимущество перед обещанием быть отмененным
        - Observable предоставляет такие операторы,как map,forEach, reduce, ... и другие операторы для работы с массивами.
        
11. Что такое Observer?
     
     Объект наблюдающий за асинхронным потоком.
     
     Observable - это название механизма, который используется в Angular для программирования асинхронных потоков данных в декларативном стиле. Observable является синонимом термина stream. К асинхронным потокам данных обычно относятся такие концепции, как события, http запросы и триггеры в коде.
     Данный механизм позволяет создавать объекты, инициирующие асинхронные потоки (stream или observable) и объекты, которые за ними наблюдают (observer).

     b) Отличия от Observable

     RxJS Observable реализуют принцип работы событий, поддерживая возможность регистрации неограниченного числа обработчиков отправляемых ими данных.
     
     a) Subject, BehaviorSubject, ReplaySubject, AsyncSubject.
     b) RxJS Subject является разновидностью объектов Observable. Особенность Subject в том, что он может отправлять данные одновременно множеству "потребителей", которые могут регистрироваться уже в процессе исполнения Subject, в то время как исполнение стандартного Observable осуществляется уникально для каждого его вызова.
     Объекты RxJS Subject реализуют принцип работы событий, поддерживая возможность регистрировать неограниченное количество обработчиков отправляемых ими данных.

16. View encapsulation:
     a) Что такое
     b) Какие типы бывают
     c) ng-deep - что это, чего стоит опасаться
     d) Компонент имеет следующий файл стилей scss. Как это может негативно сказаться на приложении? Как это исправить?
     ::ng-deep {
       .my-class {
         background: black;
       }
     }

     a) В Angular стили CSS компонентов инкапсулируются в представление компонента и не влияют на остальную часть приложения. Ангулар использует инкапсуляцию стилей компонентов. Стили элементов должны быть прописаны непосредственно в компонентах. Это дает возможность использовать в различных компонентах различные стили с одинаковым названием.
     
     b) Чтобы контролировать, как эта инкапсуляция происходит для каждого компонента, установите режим инкапсуляции представления в метаданных компонента. Выберите один из следующих режимов:
     - ShadowDom инкапсуляция представления использует встроенную реализацию теневой модели DOM браузера, чтобы прикрепить теневую DOM к основному элементу компонента, а затем помещает представление компонента внутри этой теневой DOM. Стили компонента включены в теневой DOM.
     - Emulated инкапсуляция представления (по умолчанию) имитирует поведение теневой модели DOM путем предварительной обработки (и переименования) кода CSS для эффективного применения CSS к представлению компонента. Подробнее см. Проверка сгенерированного CSS .
     - None означает, что Angular не выполняет инкапсуляцию представления. Angular добавляет CSS к глобальным стилям. Обсуждаемые ранее правила определения объема, изоляции и защиты не применяются. Этот режим по сути такой же, как вставка стилей компонента в HTML.
     Чтобы установить режим инкапсуляции компонента, используйте encapsulationсвойство в метаданных компонента.
     
     c) Отключает инкапсуляцию что дает возможность получить доступ к стилям компонента в shadowDom. Но так же отключает инкапсуляцию и делает стили глобальными.
     Применение псевдокласса ::ng-deep к любому правилу CSS полностью отключает инкапсуляцию представления для этого правила. Любой стиль с примененным ::ng-deep становится глобальным стилем. Чтобы применить указанный стиль к текущему компоненту и всем его потомкам, обязательно :host селектор : host перед ::ng-deep . Если комбинатор ::ng-deep используется без селектора псевдокласса :host , стиль может перетекать в другие компоненты.
     
     d) Так как ng-deep отключает инкапсуляцию то background может распространяться и на другие компоненты. Следует добавить селектор :host для указания того что стили будут применяться только к компоненту и его наследникам.

17. ng-content, ng-template, ng-container:
     a) Что это такое?
     b) Передача и отображение шаблонов в дочернем компоненте через templateRef и ng-content

     a) 
     - Ng-контент - это инструкция отображения контента (также называемая встраиванием содержимого). Отображение содержимого относится к встраиванию кода шаблона в компонент, что удобно для настройки повторно используемых компонентов, расширяет функции компонента и облегчает повторное использование кода.
     - ng-template директива 'отрисовывает' Angular шаблон: это означает, что содержимое этого тега будет содержать часть шаблона, которая затем может быть использована вместе с другими шаблонами для формирования окончательного шаблона компонента. Директива ng-template используется под капотом в ngIf, ngFor и ngSwitch директивах.
    - Директива ng-container позволяет применить структурную директиву к разделу страницы, не создавай при этом дополнительный элемент (тег). Итак, чтобы не создавать дополнительный div мы можем воспользоваться директивой ng-container (в разметке тега ng-container вы не увидите) и уже на ней применить структурную директиву. Есть еще важная черта директивы ng-container: она может предоставить заполнитель для инжектирования динамического шаблона на страницу.
    
    b) Для доступа к шаблону мы воспользуемся типом TemplateRef — это ссылка на элемент ng-template в вашем компоненте или директиве. У вас есть два способа получить доступ к шаблону — используя тег ng-template и Dependency Injection или используя поиск элементов через Query-декораторы.
    Элемент ng-content позволяет родительским компонентам внедрять код html в дочерние компоненты.
